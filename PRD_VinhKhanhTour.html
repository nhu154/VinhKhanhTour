Chapter 00

Giới Thiệu Sản Phẩm

VinhKhanhTour là hệ thống full-stack 3 tầng giúp du khách khám phá ẩm thực đường phố Vĩnh Khánh, Q4, TP.HCM. Phiên bản v4 bổ sung Owner Approval Workflow (chủ quán gửi yêu cầu → admin duyệt), Languages API server-side, Role-based CMS (admin/owner/user) và UserSession singleton hỗ trợ Guest mode.

Mục Tiêu

  • Thuyết minh audio tự động qua geofencing (50m radius)
  • 2-Tier TTS: Google Cloud Wavenet (online) ↔ MAUI (offline)
  • Tour theo chủ đề: Ốc, Nướng, Vặt, Đặc sản
  • Chủ quán tự quản lý nội dung qua Owner Portal
  • Admin phê duyệt mọi thay đổi (Approval Workflow)

3 Vai Trò Người Dùng

  • Admin: Full access WebCMS — CRUD tất cả, phê duyệt yêu cầu
  • Owner (Chủ quán): Quản lý quán mình, gửi yêu cầu sửa/tạo POI
  • User / Guest: Dùng Mobile app — xem bản đồ, nghe thuyết minh

Tính Năng Mới v4

  • Approval Workflow — chủ quán gửi → admin duyệt → auto apply
  • Languages API — quản lý ngôn ngữ server-side (MySQL)
  • UserSession — Guest/User/Owner mode persist qua Preferences
  • Owner Dashboard — trang riêng cho chủ quán trên WebCMS
  • Role-based Permissions — menu ẩn/hiện theo role
  • Auto-translate — dịch TTS script tự động qua Google Translate

Chapter 03

GPS & Geofencing

GPS Update→ Haversine ≤ 50m?→ Cooldown 5min OK?→ Nearest POI→ PlayCommentaryAsync

Haversine Formula

Earth radius 6,371,000m. Tính khoảng cách chính xác trên bề mặt trái đất.

Cooldown 300s

lastTriggered[id] — sau phát audio, chờ 5 phút mới trigger lại cùng POI.

Nearest Selection

Nhiều POI trong 50m → chọn gần nhất. Single audio tại 1 thời điểm.

Chapter 04

Audio TTS — 2 Tier System

Tier 1
Google Cloud TTS — Wavenet (Online)vi-VN-Wavenet-A, en-US-Wavenet-F, cmn-CN-Wavenet-A. Base64 → MP3 → Android MediaPlayer.
Tier 2
MAUI TextToSpeech (Offline)TextToSpeech.Default.SpeakAsync() — giọng built-in device. Locale matching tự động.
Dynamic Language Script: Restaurant.Translations JSON {"ja":{"name":"...","tts":"..."}} → AudioService kiểm tra currentLang → deserialize → lấy tts field. Fallback chain: target → vi → auto-generated.
Chapter 05 —

UserSession — Quản Lý Phiên

Singleton UserSession persist qua Preferences. Hỗ trợ 3 chế độ: Guest (ẩn danh), User (đã đăng nhập), Owner (chủ quán). Property IsAuthenticatedUser = IsLoggedIn && !IsGuest.

LoginAsUser()

Lưu username, fullName, isGuest=false. Profile hiển thị tên thật.

LoginAsGuest()

username="guest", fullName="Du khách". Truy cập app mà không cần đăng ký.

Logout()

Clear tất cả Preferences keys. Quay về LoginPage.

Chapter 07 —

Approval Workflow — Owner → Admin

Chủ quán gửi yêu cầu (create_poi, update_info, update_audio, update_image) → Admin duyệt/từ chối → Nếu approved, hệ thống tự động áp dụng data vào restaurants table bao gồm xử lý ảnh base64, tạo record mới hoặc cập nhật record cũ.

Owner→ POST /approvals→ Pending→ Admin Review→ Approved→ Auto Apply to DB

4 Loại Action

  • create_poi — Tạo POI mới + gán owner
  • update_info — Sửa tên, mô tả, giờ mở cửa...
  • update_audio — Sửa TTS script vi/en/zh
  • update_image — Đổi ảnh đại diện

Auto Apply Logic

Khi status = "approved", server tự động: parse RequestData JSON → xử lý base64 image → INSERT (create) hoặc UPDATE (edit) vào restaurants. Nếu create → auto INSERT user_locations.

Data Tracking

OldData lưu dữ liệu cũ trước khi sửa. RequestData chứa dữ liệu mới. Admin có thể xem diff trước khi duyệt.

Chapter 08 —

Languages API — Server-side Registry

LanguagesController tự tạo table app_languages nếu chưa có (EnsureTableAsync), seed 3 ngôn ngữ mặc định (vi/en/zh). Mobile app gọi GET /api/languages để lấy danh sách ngôn ngữ dynamic. WebCMS auto-translate TTS script qua Google Translate API miễn phí.

Auto-Create Table

CREATE TABLE IF NOT EXISTS app_languages — không cần migration thủ công. Tự seed vi/en/zh nếu bảng trống.

Protected Defaults

Không cho xóa vi/en/zh (400 BadRequest). Dynamic langs (ja, ko, fr...) có thể thêm/xóa tự do.

Auto-translate

WebCMS dùng translate.googleapis.com free API dịch TTS script từ Vietnamese sang target language tự động.

Owner Portal — Quán Của Tôi

Khi login với role=owner → WebCMS hiển thị "Quán của tôi" làm trang mặc định. Owner có thể xem stats quán mình, gửi yêu cầu sửa/tạo POI, theo dõi trạng thái duyệt.

Owner Stats

4 ô thống kê: lượt ghé, đánh giá, yêu cầu pending, yêu cầu approved. Chart 7 ngày riêng quán.

Gửi Yêu Cầu

Owner chỉnh sửa info/audio/image → POST /api/approvals → chờ admin review. Tracking real-time trạng thái.

Register Location

POST /api/approvals/register-location → liên kết user_id ↔ restaurant_id. Admin xác nhận.
